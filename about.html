What's actually running this page

Transport:raw TCP sockets
Protocol:HTTP/1.1, hand-parsed
Concurrency:one thread per connection
Framework:none
Routing:dict-based path → handler

No Flask, no Express, no http.server shortcuts. This server opens a socket, binds a port, and parses incoming bytes into a request by hand — the goal was understanding what a framework normally hides, not shipping fast.

Each client gets accepted and handed to its own thread, so one slow connection can't freeze the rest — with a socket timeout in place specifically to guard against a Slowloris-style stall.

Currently
  • threaded connection handling
  • per-connection timeout
  • static dict-based routing
  • buffered recv() for large requests
  • keep-alive via Content-Length
